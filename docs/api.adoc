
== Report Template

The file represented by `templateFilename` must exist in the template folder of the `dina-export-api`.

[source, json]
----
{
  "data": {
    "type": "report-template",
    "attributes": {
      "group": "aafc",
      "templateOutputMediaType": "text/html",
      "outputMediaType": "application/pdf",
      "includesBarcode": true,
      "name": "demo-label",
      "templateFilename": "demo.ftlh"
    }
  }
}
----

== Report Request
[source, json]
----
{
  "data": {
    "type": "report-request",
    "attributes": {
      "group": "aafc",
      "reportTemplateUUID": "08b871ed-52c3-4ef9-be86-a51504499a21",
      "payload": {
        "elements": [
          {
            "barcode": {
              "id": "my-id",
              "content": "ABC-209393"
            },
            "data": {
              "attributes": {
                "materialSampleName": "ABC-209393"
              }
            }
          }
        ]
      }
    }
  }
}
----

Where the report template would be

[source, html]
----
<html>
  <head>
    <title>Test QR Code</title>
  </head>
  <body>
    <div>
      <img src="${elements[0].barcode.id}.png" width="100" height="100" />
    </div>
    <div>${elements[0].data.attributes.materialSampleName}</div>
  </body>
</html>
----

== Data Export Request

Data Exports run asynchronously. Upon creation using POST, an export UUID is returned. Use this UUID to query the export status with a GET request to the same endpoint (`data-export`).

=== Single-Entity Export

For a single entity type, the export generates a single CSV/TSV file. Use the `schema` attribute to define entity types (kebab-case) and their fields:

POST /data-export
[source, json]
----
{
  "data": {
    "type": "data-export",
    "attributes": {
      "name": "Material Sample Export",
      "source": "dina_material_sample_index",
      "query": "{\"query\": {\"match_all\": {}}}",
      "schema": {
        "material-sample": [
          "id",
          "materialSampleName",
          "collectingEvent.dwcVerbatimLocality"
        ]
      },
      "exportOptions": {
        "columnSeparator": "COMMA"
      }
    }
  }
}
----

=== Multi-Entity Export (ZIP Package)

For multiple entity types, enable packaging to generate separate CSV files packaged in a ZIP. Each entity type gets its own file (e.g., `material-sample.csv`, `collecting-event.csv`):

[source, json]
----
{
  "data": {
    "type": "data-export",
    "attributes": {
      "name": "Multi-Entity Export",
      "source": "dina_material_sample_index",
      "query": "{\"query\": {\"match_all\": {}}}",
      "schema": {
        "material-sample": [
          "id",
          "materialSampleName",
          "group"
        ],
        "collecting-event": [
          "id",
          "dwcVerbatimElevation",
          "createdBy"
        ]
      },
      "columnAliases": {
        "material-sample": [
          "Sample ID",
          "Sample Name",
          "Group"
        ],
        "collecting-event": [
          "Event ID",
          "Elevation",
          "Created By"
        ]
      },
      "exportOptions": {
        "enablePackaging": "true",
        "enableIdTracking": "true",
        "columnSeparator": "COMMA"
      }
    }
  }
}
----

NOTE: Entity type keys use kebab-case (e.g., `material-sample`, `collecting-event`). The `columnAliases` count must match the schema column count for each entity.

=== Export Options

* `columnSeparator`: `COMMA` (default) or `TAB`
* `enablePackaging`: `true` creates separate files per entity type in a ZIP (requires multiple entities in schema)
* `enableIdTracking`: `true` prevents duplicate related entities across files in multi-entity exports

=== Functions

Functions create calculated columns. Add the function name to your schema and define it in `functions`:

[source, json]
----
{
  "schema": {
    "material-sample": ["id", "materialSampleName", "latLong"]
  },
  "functions": {
    "latLong": {
      "functionDef": "CONVERT_COORDINATES_DD",
      "params": {
        "column": "collectingEvent.eventGeom"
      }
    }
  }
}
----

Available functions:

* `CONVERT_COORDINATES_DD`: Converts geometry to decimal degrees format
** Parameters: `column` (field path to geometry)

* `CONCAT`: Combines multiple fields with a separator
** Parameters: `items` (list of field names), `constants` (map of constant values), `separator` (delimiter)

[source, json]
----
{
  "functions": {
    "link": {
      "functionDef": "CONCAT",
      "params": {
        "items": ["url", "id"],
        "constants": {"url": "http://localhost/entry"},
        "separator": "/"
      }
    }
  }
}
----

=== Deprecated Format (columns)

The `columns` attribute is deprecated and will be removed in version 0.19. Use `schema` instead

== Download File

For data exports, the `fileId` is the UUID returned by the export request.

[source]
----
GET /file/{fileId}?type=LABEL|DATA_EXPORT
----
